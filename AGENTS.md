# AGENTS.md

## 역할

당신은 `living-aegis-simulator` 저장소에서 작업하는 Codex입니다.

이 저장소는 Living Aegis Origin 제작을 위한 simulator 모음입니다. 메인 게임 저장소도 아니고, 게임 기능 prototype 저장소도 아닙니다. 작업을 시작하기 전에 먼저 저장소의 기준 문서를 읽고 현재 상태를 파악하세요.

## 먼저 확인할 문서

작업 전, 아래 문서를 우선 확인하세요.

- `README.md`
- `SIMULATION_INDEX.md`
- `SIMULATION_NOTES.md`
- `DECISION_LOG.md`
- `CHANGELOG.md`
- `docs/` 안의 관련 문서
- 각 simulator 폴더의 `README.md`

## 작업 원칙

- 기존 문서와 현재 커밋 상태를 기준으로 작업하세요.
- 파일명, 폴더명, 코드 식별자는 임의로 바꾸지 마세요.
- 기존 simulator 기능을 깨뜨리지 마세요.
- 대규모 리팩터링은 하지 마세요.
- 한 번에 하나의 작은 작업만 수행하세요.
- 각 simulator는 가능한 한 독립 폴더에서 실행 가능하게 유지하세요.
- simulator 폴더명에는 순번을 붙이지 마세요.
- 루트 `index.html`은 simulator 런처 페이지 역할을 유지하세요.
- 내부 설명 문서는 한국어 중심으로 작성하세요.
- 기술 용어, 파일명, 코드명은 영어를 유지해도 됩니다.
- 작업 범위 밖의 파일은 수정하지 마세요.
- 변경이 필요한 경우, 관련 문서도 함께 갱신하세요.
- 작업 결과를 요약할 때는 수정한 파일과 확인한 파일을 구분해서 알려주세요.

## 문서 작성 원칙

- 사용자가 반복해서 읽는 설명 문서는 한국어로 작성하세요.
- 기술 용어는 필요하면 영어를 유지하고, 한국어 설명을 함께 붙이세요.
- `README.md`, `SIMULATION_INDEX.md`, `SIMULATION_NOTES.md`, `DECISION_LOG.md`, `CHANGELOG.md`는 현재 상태와 어긋나지 않게 유지하세요.
- `docs/` 안의 문서는 시뮬레이션에서 얻은 기준, 관찰, 해석을 한국어 중심으로 기록합니다.
- 각 simulator 폴더의 `README.md`는 해당 시뮬레이터의 목적과 현재 상태를 간단히 설명해야 합니다.
- 이 파일은 항상 적용되는 저장소 작업 원칙이며, 특정 대화에서만 적용되는 상태 확인 지시와 구분하세요.

## 금지 사항

- 사용자의 명시적 요청 없이 파일명이나 폴더명을 변경하지 마세요.
- 사용자의 명시적 요청 없이 프로젝트 구조를 크게 바꾸지 마세요.
- 사용자의 명시적 요청 없이 새 simulator를 추가하지 마세요.
- 사용자의 명시적 요청 없이 기능을 추가하지 마세요.
- 문서 정리 작업 중에는 코드 동작을 바꾸지 마세요.
- 메인 게임 구현을 이 저장소에서 진행하지 마세요.
- prototype 성격의 게임 기능 실험을 이 저장소에 추가하지 마세요.
